## Problem to fix

**Royal Diamonds are being handled as Diamonds instead of Royals.**

According to the approved beginner rules, **all Jacks, Queens, and Kings are Royals regardless of suit**, including:

* Jack of Diamonds
* Queen of Diamonds
* King of Diamonds

So `J♦`, `Q♦`, and `K♦` must be treated as **Royals**, not as regular Diamond cards.

## Important rule

When a card is both:

* suit = `D`
* and `isRoyal = true`

it must follow **Royal behavior first**, not Diamond behavior.

That means Royal Diamonds should:

* be playable as Royals
* use Royal actions
* go to Court as Royals
* not get normal Diamond actions like:

  * play to Mine
  * discard to draw
  * discard for boost

## Likely current bug

Please check for any logic that branches by suit first, especially patterns like:

```ts
if (card.suit === "D") { ... }
if (card.isRoyal) { ... }
```

If that exists, it should be changed so Royals are checked first, for example:

```ts
if (card.isRoyal) { ... }
if (card.suit === "D") { ... }
```

or equivalent logic that ensures Royal Diamonds are treated as Royals.
