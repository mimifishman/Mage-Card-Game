  * This already seems to exist. Keep it working.
* **Return:** Discard Spade to Abyss, choose a card in the Abyss with value **less than or equal to** the Spade’s value, and return that chosen card to the acting player’s hand

### 3) `artifacts/api-server/src/game/attachments.ts` or a new focused file if cleaner

Implement the missing backend logic.

#### A. Heart discard to heal player

Create a function like:

```ts
discardHeartToHeal(state, playerId, heartCardId, targetPlayerId)
```

Required behavior:

* validate it is the acting player’s turn / card can be played using the repo’s existing validation style
* validate the card is a **non-Royal Heart**
* remove the Heart from the acting player’s hand
* put the Heart into `state.abyss`
* increase the target player’s life by the Heart’s value
* return updated game state

Keep the validation and state-update style consistent with the rest of the codebase.

#### B. Spade discard to return from Abyss

Create a function like:

```ts
discardSpadeToReturn(state, playerId, spadeCardId, targetCardId)
```

Required behavior:

* validate it is playable using the repo’s existing validation style
* validate the card is a **non-Royal Spade**
* validate `targetCardId` is currently in the Abyss
* validate the target card’s value is **less than or equal to** the Spade’s value
* remove the Spade from the acting player’s hand
* move the Spade to the Abyss
* remove the selected target card from the Abyss
* add the selected target card to the acting player’s hand
* return updated game state

Important:

* the Spade itself should remain in the Abyss after use
* the selected card should leave the Abyss and go to the player’s hand
